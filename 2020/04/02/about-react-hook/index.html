初識React Hook與useState

Fatz's Record
2020-04-02
react
react, react hook

大約有半年的時間我都在準備研究所的考試，最近跟室友聊天，他說現在React都在用Hook了，我才真正體會到，前端的世界變得這麼快！
自己半年前在研究React時，Hook只是個剛出來不久的東西，當時我保持著觀望的態度沒有好好的了解，現在卻變成了討論的熱門話題，甚至可以說是React未來的趨勢，所以我也趁著最近接了新的專案，更新了一下技術，果然不讓我失望！

這之前我所知道的React

在React Hook出來之前，Component主要分成兩類：

  • stateful component (class component)
  • stateless component (function component)

一般認為stateful就是class形態的component，這類的component顧名思義保存著一些狀態，並且在使用this.setState進行狀態更新時可以觸發重新渲染畫面，也可以搭配Lifecycle Hooks(這個Hook不是在說React Hook)在生命週期中加入一些邏輯、side effect等等。
而stateless就是function形態的component，這類的component其行為比較單純，沒有自己的狀態，主要就是接收外界傳入的props，再根據props去渲染畫面，比較偏向是專門用來顯示UI的組件。
為了區別這兩種不同的component，並且集中管理網站的狀態，有些專案會將function component放在components資料夾，class component放在containers資料夾，形成以下結構：

React Hook為我帶來的好處

我甚至不需要寫任何class component就能完成一個專案，React Hook讓function component能夠管理自己的狀態、以及模擬class component常用的生命週期鉤子(componentDidMount, componentDidUpdate, etc…)，如果沒有特殊的需求，這樣真的就很夠用！

我想已經回不去了

React Hook的出現確實是一個很大的進化，單就自己在專案節省的時間成本來看，效益真的很大，而React生態圈的各個工具也紛紛開始支援Hook的概念，像是React Router提供了useParam、useHistory，讓function component可以接收夾帶在url的parameters、參考瀏覽器history物件等；Redux提供了useDispatch、useSelector取代以前煩雜的mapXXXToProps系列用法，甚至useContextHook的出現引發了社群對於Redux是否還有存在必要的熱烈討論，那又是另外的故事。
